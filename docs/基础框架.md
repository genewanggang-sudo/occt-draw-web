# 基础框架

## 文档目的

本文记录当前仓库的基础框架状态，作为后续实现、评审和协作时的共同参考。

当前项目不再以二维出图编辑器为目标，而是面向三维云端 CAD 工作台组织。阶段节奏以 [路线图](./路线图.md) 为准。

## 当前基础设施

仓库采用轻量 `pnpm workspace` monorepo：

- `apps/web`：React + Vite 浏览器工作台
- `packages/*`：按领域和基础能力拆分的内部包
- `docs`：中文工程文档
- `mise.toml`：当前项目专用 Node 和 pnpm 版本管理

当前项目内运行时版本：

- Node.js：`24.15.0`
- pnpm：`10.33.2`

这些版本只对当前项目生效，不改变其它项目的全局默认版本。

## 当前实现状态

旧的二维出图实现已经删除，包括二维图元模型、Canvas 2D 渲染器和前端绘制交互。

现在已经具备三维云端 CAD 的工作台骨架：

- 包边界已经重新规划
- 依赖方向已经重新约束
- 页面层已经具备顶部栏、命令工具栏、视图工具栏、模型树、属性面板和三维视窗
- `core` 已有默认文档、PartStudio、对象、选择目标和特征占位
- `math` 已有三维向量和基础数学函数
- `scene` 已能把 CAD 对象映射为渲染场景对象
- `renderer` 已有相机、包围盒、包围球、拾取和渲染输入抽象
- `renderer-webgl` 已有 WebGL 线框渲染后端
- 视窗已支持旋转、平移、缩放、适配视图和标准视图
- 选择链路已支持调试立方体边拾取、预选、高亮和属性面板展示
- Wasm 仍作为未来几何内核入口，目前只保留桥接包边界

## 下一阶段建议

当前处于路线图第 1 阶段，建议按下面顺序推进：

1. 补齐命令生命周期和命令 UI 状态。
2. 稳定选择系统边界，让对象选择、子元素选择、清空选择和命令消费选择走统一控制器。
3. 建立撤销重做最小模型，先覆盖命令状态和选择变化。
4. 固化 `core` 文档、PartStudio、Feature、CadObject 的最小模型和不可变更新方式。
5. 保持 `scene` 只做领域对象到渲染对象的映射，不承载业务状态。
6. 第 1 阶段完成后，再进入草图系统 v1。
