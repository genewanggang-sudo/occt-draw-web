# 基础框架设计

## 文档目的

本文记录当前仓库的基础框架状态，作为后续实现、评审和协作时的共同参考。

当前项目不再以二维出图编辑器为目标，而是面向三维云端 CAD 工作台重新组织。

## 当前基础设施

仓库采用轻量 `pnpm workspace` monorepo：

- `apps/web`：React + Vite 浏览器工作台
- `packages/*`：按领域和基础能力拆分的内部包
- `docs`：中文工程文档
- `mise.toml`：当前项目专用 Node 和 pnpm 版本管理

当前项目内运行时版本：

- Node.js：`24.15.0`
- pnpm：`10.33.2`

这些版本只对当前项目生效，不改变其它项目的全局默认版本。

## 当前状态

旧的二维出图实现已经删除，包括二维图元模型、Canvas 2D 渲染器和前端绘制交互。

现在保留的是三维云端 CAD 的框架骨架：

- 包边界已经重新规划
- 依赖方向已经重新约束
- 页面层只保留占位工作台
- Wasm 仍作为未来几何内核入口
- WebGL 渲染后端已预留独立包

## 下一阶段建议

建议按下面顺序推进：

1. 完成 `math` 第一版三维数学基础。
2. 完成 `scene` 第一版场景图和对象树。
3. 完成 `renderer` 渲染抽象接口。
4. 完成 `renderer-webgl` 最小三维视窗。
5. 完成 `core` 文档、命令、选择和历史边界。
6. 开始 `sketch`、`constraints`、`parametrics` 的最小模型。
7. 再接入 `wasm-bridge` 和真实 `occt-draw-core` Wasm。
