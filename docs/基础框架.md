# 基础框架

## 文档目的

本文记录当前仓库的基础框架状态，作为后续实现、评审和协作时的共同参考。

当前项目不再以二维出图编辑器为目标，而是面向三维云端 CAD 工作台。

## 当前基础设施

- `apps/web`：React + Vite 浏览器工作台。
- `packages/*`：按领域和基础能力拆分的内部包。
- `docs`：中文工程文档。
- `mise.toml`：当前项目专用 Node 和 pnpm 版本管理。

当前项目运行时版本：

- Node.js：`24.15.0`
- pnpm：`10.33.2`

这些版本只对当前项目生效，不改变其它项目的全局默认版本。

## 当前实现状态

- 旧的二维出图实现已经删除，包括二维图元模型、Canvas 2D 渲染器和前端绘制交互。
- 包边界已经重新规划，依赖方向已经通过 dependency-cruiser 约束。
- 页面层已经具备顶部栏、命令工具栏、视图工具栏、模型树、属性面板和三维视窗。
- `math` 已改为 class-based CAD 几何基础，提供 `Vec3`、`Point3`、`Vec2`、`Point2`、`Plane`、`Ray3`、`LineSegment3`、`Matrix4`、`Transform3` 等对象。
- `core` 已具备 `CadDocument`、`PartStudio`、对象、选择、特征占位、`EditDraft`、`DocumentOperation`、`DocumentTransaction` 和 `TransactionGroup`。
- `display` 负责把 `CadDocument + EditDraft` 投影为 `DisplayModel`，替代原来的 `scene` 包。
- `renderer` 已有相机、包围盒、包围球、拾取和渲染输入协议。
- `renderer-webgl` 已有 WebGL 线框渲染后端。
- 视窗支持旋转、平移、缩放、适配视图和标准视图。
- 选择链路支持调试立方体边拾取、预选、高亮、模型树联动和属性面板展示。
- 命令层已改为 `CadCommand class` 入口，`SelectCommand` 负责选择，`SketchCommand` 作为下一阶段占位入口。
- 当前尚未新增独立 `editor` 包，命令、交互和编辑器状态流仍主要位于 `apps/web/src/editor`，下一步应先收口到 `packages/editor`。
- Wasm 仍作为未来几何内核入口，目前只保留桥接包边界。

## 下一阶段建议

当前仍处于第 1 阶段收口。下一步进入草图系统 v1 前，只允许继续补齐必要架构边界，不继续扩展工作台壳功能。

推荐顺序：

1. 新增并收口 `packages/editor`，承接 UI 之外的命令、交互、选择、视图状态和编辑器状态流。
2. 确认 `math/core/editor/display/renderer` 边界稳定。
3. 进入草图系统 v1，先做默认草图平面和线段绘制最小闭环。
4. 草图线段先进入 `EditDraft`，确认后生成 `DocumentOperation`，再由 `display` 显示正式对象。
