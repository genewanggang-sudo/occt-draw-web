# 三维云端 CAD 产品路线图

本文档用于约束 `occt-draw-web` 的长期产品开发节奏，避免在局部实现细节中偏离“三维云端 CAD”主线。

## 总体目标

`occt-draw-web` 的目标是构建一个运行在浏览器中的三维云端 CAD：

- 前端应用使用 React + Vite。
- 渲染引擎使用 WebGL，并逐步演进为独立可替换的渲染后端。
- 草图、约束、命令系统、应用状态和文档模型优先使用 TypeScript 实现。
- 几何内核通过 Wasm 接入 `occt-draw-core`。
- 后续扩展云端文档、版本历史、权限、协同和工程化产品能力。

## 当前阶段

当前处于 **第 0 阶段：三维云端 CAD 前端技术底座收口**。

本阶段不是正式建模功能开发期，而是确认工程底座、渲染链路、相机控制、包边界和代码分层是否能够支撑长期开发。

## 第 0 阶段：技术底座收口

目标：让项目具备长期开发的工程基础，不在后续功能迭代中快速失控。

已完成内容：

- Monorepo、pnpm、TypeScript、依赖边界和基础校验链路。
- WebGL 最小三维视窗。
- 正交相机、网格、坐标轴、立方体线框。
- 旋转、平移、缩放、适配视图和标准视图。
- `renderer`、`renderer-webgl`、`apps/web` 的初步包内分层。

剩余重点：

- 应用状态骨架。
- 命令系统骨架。
- CAD 文档最小模型。
- 选择系统之前的状态边界设计。

完成标准：

- 能稳定表达“文档 + 视窗 + 命令入口”。
- UI 不追求完整，但代码结构可以承载后续功能。
- 不在本阶段追求真实建模能力。

## 第 1 阶段：CAD 产品骨架

目标：从“一个三维视窗”变成“一个 CAD 工作台”。

核心内容：

- `Document`
- `PartStudio`
- `Scene`
- `Selection`
- `Command`
- `FeatureTree` 占位
- 顶部工具区
- 左侧模型树占位
- 右侧属性或命令面板占位

完成标准：

- 页面具备真实 CAD 产品的基本工作台结构。
- 能打开一个默认文档。
- 能看到模型树中的对象。
- 能进入和退出命令状态。
- 暂不做真实建模。

## 第 2 阶段：选择与交互系统

目标：让用户能够和三维对象产生基础交互。

核心内容：

- 左键选择。
- 框选基础。
- hover 高亮。
- selection state。
- 命令和选择联动。
- 后续 WebGL 拾取能力的基础接口。

完成标准：

- 用户能点选默认场景中的对象。
- UI 能显示当前选中对象。
- 命令系统能够消费选择结果。

## 第 3 阶段：草图系统 v1

目标：让 CAD 进入“可创建几何”的阶段。

核心内容：

- 草图平面。
- 点、线、圆、圆弧。
- 草图状态机。
- 鼠标绘制。
- 草图对象数据结构。
- 简单尺寸和约束占位。
- 草图进入和退出流程。

完成标准：

- 可以在一个平面上画基础草图几何。
- 草图数据进入文档模型。
- 渲染层能显示草图几何。
- 暂不追求复杂约束求解。

## 第 4 阶段：约束系统 v1

目标：让草图从“画图”逐步变成“CAD 草图”。

核心内容：

- 重合约束。
- 水平和垂直约束。
- 平行和垂直约束。
- 等长约束。
- 距离、半径、角度约束。
- TypeScript 约束求解器雏形。

完成标准：

- 草图对象可以被简单约束驱动。
- 拖动点时基础约束能够保持。
- 架构能够继续扩展，而不是一次性追求工业级求解器。

## 第 5 阶段：Wasm 几何内核接入

目标：让草图能够生成三维实体。

核心内容：

- `occt-draw-core` Wasm bridge。
- profile 识别。
- 拉伸。
- 基础 BRep 或 mesh 返回。
- worker 化。
- 内核错误处理。

完成标准：

- 画一个封闭草图。
- 执行拉伸。
- 生成一个三维实体。
- WebGL 能渲染实体边线和面片。

## 第 6 阶段：特征建模 v1

目标：形成 CAD 的核心建模链路。

核心内容：

- Feature timeline。
- Sketch feature。
- Extrude feature。
- Boolean add / cut / new body。
- 参数编辑。
- 特征重算。
- 失败状态显示。

完成标准：

- 用户可以创建草图、拉伸、修改参数并重新计算。
- 左侧特征树能反映建模历史。
- 建模过程具备最小可用闭环。

## 第 7 阶段：文件与云端文档

目标：从本地演示进入云端 CAD 产品形态。

核心内容：

- 本地序列化格式。
- 文档保存和重新打开。
- 云端项目空间。
- 自动保存。
- 版本历史。
- 权限模型。
- 后端 API。

完成标准：

- 用户能创建、保存、重新打开一个 CAD 文档。
- 文档数据不再只存在内存中。
- 具备后续协同能力的文档边界。

## 第 8 阶段：协同与工程化产品能力

目标：靠近真正的云端 CAD 产品。

核心内容：

- 多人协同状态。
- 评论和标注。
- 版本对比。
- 分享链接。
- 权限控制。
- 云端计算任务入口。
- 性能优化。

完成标准：

- 多用户、多设备可以围绕同一模型工作。
- 产品具备云端 CAD 的基础协作形态。

## 节奏控制原则

- 每个阶段只解决一个核心问题。
- 当前阶段没有完成前，不提前堆叠后续复杂功能。
- 技术底座服务产品目标，不能为了抽象而抽象。
- UI 先服务核心流程，不在早期过度设计工作台细节。
- Wasm 几何内核只负责高精度几何和实体建模，不承担应用状态、命令系统和草图交互。
- 如果某个任务无法归入当前阶段，应先暂停并重新确认是否会让产品路线跑偏。
