# 文档索引

本目录存放 `occt-draw-web` 的中文工程文档。当前项目方向是三维云端 CAD，后续开发应优先服从路线图，再进入具体架构和工程实现。

## 阅读顺序

1. [路线图](./路线图.md)
   说明当前实现到哪一步、当前正在做什么、每个后续阶段要实现什么、完成标准是什么，以及哪些内容不能提前做。

2. [架构设计](./架构设计.md)
   说明产品定位、技术路线、分包结构、依赖方向、Wasm 边界和云端能力边界。

3. [基础框架](./基础框架.md)
   说明当前仓库基础设施、已具备的工作台能力和近期工程收口项。

4. [工程规范](./工程规范.md)
   说明 TypeScript、导入、包边界、提交、校验、版本管理和路线图约束。

5. [运行时约定](./运行时约定.md)
   说明环境变量、Wasm 静态资源、WebGL 资源、Worker 调度和云端接口边界。

## 当前阶段

当前处于 [路线图](./路线图.md) 中的 **第 1 阶段：CAD 工作台骨架与基础交互**。

已经具备三维视窗、默认文档、模型树、属性面板、命令入口、视图导航、WebGL 线框渲染和基础选择高亮。接下来应优先补齐命令生命周期、选择边界、撤销重做最小模型和 `core` 文档模型稳定性。

## 维护规则

- 文档统一使用中文。
- 新增设计决策时同步更新索引。
- 路线图是开发主线，新增功能应先确认属于哪个阶段。
- 与三维云端 CAD 方向冲突的旧二维出图内容应及时删除或改写。
- 代码里的类型名、变量名和包名保持英文，文档和产品文案使用中文。
