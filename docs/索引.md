# 文档索引

本目录存放 `occt-draw-web` 的中文工程文档。当前项目方向是三维云端 CAD，后续开发应优先服从路线图，再进入具体架构和工程实现。

## 阅读顺序

1. [开发上下文交接](./开发上下文交接.md)  
   给新对话、新工具或其它开发位置快速了解当前项目状态、已确认决策、不要做的事和下一步建议。

2. [路线图](./路线图.md)  
   说明当前实现到哪一步、当前正在做什么、后续阶段目标和完成标准。

3. [架构设计](./架构设计.md)  
   说明产品定位、分包职责、依赖方向、`EditDraft / Operation / display` 等关键架构边界。

4. [math 包架构设计](./math包架构设计.md)
   说明 `math` 面向专业 CAD 草图的几何抽象、架构分层、数据流和演进方案。

5. [WebGL 引擎架构设计](./webgl引擎架构设计.md)
   说明 WebGL 渲染引擎的可复用边界、通用 RenderScene 协议方向和 CAD 适配层职责。

6. [基础框架](./基础框架.md)  
   说明当前基础设施、包结构、工程护栏和当前实现状态。

7. [工程规范](./工程规范.md)  
   说明 TypeScript、导入、包边界、校验、版本管理和路线图约束。

8. [运行时约定](./运行时约定.md)  
   说明环境变量、Wasm 静态资源、WebGL 资源、Worker 调度和云端接口边界。

## 当前阶段

当前处于 **第 1 阶段收口，准备进入第 2 阶段草图系统 v1**。

已经具备三维视窗、默认文档、模型树、属性面板、命令入口、视图导航、WebGL 线框渲染和基础选择高亮。接下来优先确认 `math/core/display/renderer/command` 边界稳定，然后进入默认草图平面和线段绘制最小闭环。

## 维护规则

- 文档统一使用中文。
- 新增设计决策时同步更新索引。
- 路线图是开发主线，新增功能应先确认属于哪个阶段。
- 与三维云端 CAD 方向冲突的旧二维出图内容应及时删除或改写。
- 代码里的类型名、变量名和包名保持英文，文档和产品文案使用中文。
