# 工程规范

## 基础原则

本仓库面向三维云端 CAD，工程规范优先保证长期可维护、包边界清晰、类型安全和多人协作稳定。

文档和产品文案使用中文；代码里的类型名、变量名、函数名和包名保持英文。

## 工具链

- Node.js：`24.15.0`
- pnpm：`10.33.2`
- 项目级版本管理：`mise`
- 页面框架：React
- 构建工具：Vite
- 语言：TypeScript strict
- 格式化：Prettier
- Lint：ESLint flat config
- 依赖方向检查：dependency-cruiser
- 提交门禁：husky + lint-staged + commitlint

## TypeScript 规则

- 禁止使用 `any`。
- 类型覆盖率必须保持 `100%`。
- 优先使用明确领域类型，不用松散对象长期承载业务状态。
- 仅类型依赖必须使用 `type import`。
- workspace 包不使用默认导出。
- 禁止 `export *`，所有公共 API 必须在 `src/index.ts` 显式导出。
- 新写源码统一使用 TypeScript `private`，不新增 `#private`。

## 导入规则

- 跨包导入只能走包入口，例如 `@occt-draw/math`。
- 禁止 deep import，例如 `@occt-draw/core/src/foo`。
- 禁止相对路径跨包导入，例如 `../../packages/foo/src`。
- 包内文件可以使用相对导入，但不能形成循环依赖。

## 包边界

- `shared`：最底层通用常量和稳定工具，不依赖其它 workspace 包。
- `math`：CAD 前端几何基础，只依赖 `shared`。
- `core`：CAD 文档、选择、EditDraft、Operation、Transaction 和基础领域模型。
- `editor`：编辑器运行时包，承接命令、交互、选择、视图状态和编辑器状态流，不依赖 React。
- `display`：把 `CadDocument + EditDraft` 投影为 `DisplayModel`。
- `renderer`：相机、包围盒、拾取、渲染输入和高亮协议。
- `renderer-webgl`：WebGL 绘制实现。
- `apps/web`：应用装配层，可以组合运行时包，但不能把业务规则散落在 React 组件里。


## Wasm 与 TypeScript 分工

TypeScript 负责编辑器状态和交互逻辑：

- 命令系统
- 选择系统
- 草图工具流
- 约束数据模型
- 参数表达式
- 文档编辑事务
- 显示模型投影

Wasm 负责几何内核能力：

- 高精度几何计算
- 拓扑生成
- 布尔运算
- 曲线曲面计算
- 模型网格化或几何输出

## 校验命令

根命令是 CI 和本地校验的统一入口：

```bash
pnpm check
```

其中包含：

- `pnpm lint`
- `pnpm typecheck`
- `pnpm type-coverage`
- `pnpm format:check`
- `pnpm depcruise`
- `pnpm build`
