# 工程规范

## 基础原则

本仓库面向三维云端 CAD，工程规范优先保证长期可维护、包边界清晰、类型安全和多人协作稳定。

文档和产品文案使用中文，代码中的类型名、变量名、函数名、包名保持英文。

新增功能必须先对照 [路线图](./路线图.md)。如果任务不属于当前阶段，应先修改路线图或明确推迟，不能在局部实现里偷偷跨阶段堆功能。

## 工具链

- Node.js：`24.15.0`
- pnpm：`10.33.2`
- 项目级版本管理：`mise`
- 前端框架：`React`
- 构建工具：`Vite`
- 语言：TypeScript strict
- 格式化：Prettier
- Lint：ESLint flat config
- 依赖方向检查：dependency-cruiser
- 提交门禁：husky + lint-staged + commitlint

`mise` 只对当前项目生效，不改变其它项目的全局 Node 和 pnpm 版本。

## TypeScript 规则

- 禁止使用 `any`。
- 类型覆盖率必须保持 `100%`。
- 优先使用明确的领域类型，不使用松散对象长期承载业务状态。
- 仅类型依赖必须使用 `type import`。
- workspace 包不使用默认导出，除非有非常明确的理由。
- 禁止 `export *`，所有公共 API 必须在 `src/index.ts` 中显式导出。

## 导入规则

- 跨包导入只能走包入口，例如 `@occt-draw/math`。
- 禁止 deep import，例如 `@occt-draw/core/internal/foo`。
- 禁止相对路径跨包导入，例如 `../../packages/foo/src`。
- 共享能力优先放在 `shared` 或更明确的底层包中，不在业务包之间复制。

## 包边界

基础包：

- `shared`：最底层公共类型、常量和工具，不依赖其它 workspace 包。
- `math`：三维数学、几何基础量、数值容差和坐标计算，只能依赖 `shared`。
- `protocol`：Worker、Wasm 和云端任务协议，只能依赖 `shared`。

领域包：

- `core`：CAD 文档、命令、选择、历史、装配和特征关系。
- `sketch`：草图平面、草图元素、草图工具流。
- `constraints`：草图几何约束、尺寸约束和约束管理。
- `parametrics`：参数、表达式、变量表和特征驱动关系。
- `scene`：三维场景图、对象树、选择集、可见性和显示状态。

运行时包：

- `renderer`：渲染抽象接口。
- `renderer-webgl`：WebGL 渲染后端。
- `wasm-bridge`：`occt-draw-core` Wasm 的唯一桥接层。
- `worker-client`：主线程侧 Worker 调用封装。
- `worker-runtime`：Worker 侧任务调度和内核调用编排。
- `cloud-client`：云端项目、文件、版本、权限和协同接口。

应用和 UI：

- `apps/web`：React + Vite 工作台，只做应用装配。
- `ui`：自有 CAD 工作台组件库。

## 依赖方向

依赖方向由 `dependency-cruiser` 强制检查：

- `shared` -> 无 workspace 依赖
- `math` -> `shared`
- `protocol` -> `shared`
- `core` -> `math`, `shared`
- `sketch` -> `core`, `math`, `shared`
- `constraints` -> `sketch`, `core`, `math`, `shared`
- `parametrics` -> `core`, `shared`
- `scene` -> `core`, `math`, `shared`
- `renderer` -> `scene`, `math`, `shared`
- `renderer-webgl` -> `renderer`, `scene`, `math`, `shared`
- `wasm-bridge` -> `protocol`, `shared`
- `worker-client` -> `protocol`, `shared`
- `worker-runtime` -> `wasm-bridge`, `protocol`, `shared`
- `cloud-client` -> `protocol`, `shared`
- `apps/web` -> 可组合运行时包

## Wasm 和 TypeScript 分工

TypeScript 负责编辑器侧状态和交互逻辑：

- 草图
- 约束
- 参数
- 命令系统
- 撤销重做
- 选择集
- 场景显示状态

Wasm 负责几何内核能力：

- 高精度几何计算
- 拓扑生成
- 布尔运算
- 特征求值
- 曲线曲面计算
- 模型网格化或几何输出

## 脚本规范

根命令是 CI 和本地校验的统一入口：

- `pnpm lint`
- `pnpm typecheck`
- `pnpm type-coverage`
- `pnpm format:check`
- `pnpm depcruise`
- `pnpm build`
- `pnpm check`

修改共享包、依赖规则、构建配置或公共文档后，应运行 `pnpm check`。

## 路线图约束

- 开发任务必须标明所属阶段。
- 当前阶段的完成标准优先于局部功能完整度。
- 可以做后续阶段的接口预留，但预留必须有明确调用边界，不能把未来业务提前写进页面组件。
- 代码实现如果改变了阶段完成度，必须同步更新 `docs/路线图.md` 的当前实现快照。
- 包边界变化必须同步更新 `docs/架构设计.md` 和 dependency-cruiser 配置。
