# 运行时与资源约定

## 环境变量

前端环境变量由 `apps/web/.env.*` 管理。

约定：

- 只有 `VITE_` 前缀变量可以暴露到浏览器。
- 不允许把密钥、令牌、私有地址放进 `VITE_` 变量。
- 示例变量维护在 `apps/web/.env.example`。

当前预留变量：

- `VITE_APP_TITLE`：前端工作台标题。
- `VITE_OCCT_WASM_ENTRY`：`occt-draw-core` Wasm 入口地址。

## Wasm 资源

`occt-draw-core` 的浏览器静态资源暂时由 `apps/web/public/wasm/` 承载。

原则：

- 页面层不直接初始化 Wasm。
- Wasm 加载和调用细节只进入 `wasm-bridge`。
- 重计算任务通过 Worker 调度。
- 未来可以替换为云端下发、版本化资源或 CDN 资源。

推荐调用链：

```txt
apps/web
  -> worker-client
    -> worker-runtime
      -> wasm-bridge
        -> occt-draw-core Wasm
```

## WebGL 资源

三维渲染资源后续由 `renderer-webgl` 管理。

预期资源包括：

- shader
- mesh buffer
- edge buffer
- selection buffer
- material preset
- viewport state

这些资源不直接散落在 React 页面组件中。

## 云端接口

云端项目、文件、版本、权限、协同和任务状态统一从 `cloud-client` 进入。

页面组件不直接拼接云端 API 路径，也不直接维护云端协议模型。
