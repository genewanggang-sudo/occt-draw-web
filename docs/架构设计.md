# 三维云端 CAD 架构设计

## 项目定位

`occt-draw-web` 的目标已经调整为三维云端 CAD 前端工作台。

当前仓库负责浏览器侧工作台、三维场景、草图交互、参数和约束的 TypeScript 逻辑、WebGL 渲染、Worker 调度、Wasm 几何内核桥接，以及未来云端项目协作能力的前端边界。

几何内核仍然由 Wasm 提供，前端不直接实现底层 B-Rep、布尔、倒角、圆角等高精度几何算法。

## 核心技术路线

- 页面框架：`React`
- 构建工具：`Vite`
- 包管理：`pnpm workspace`
- 项目级运行时管理：`mise`
- 三维渲染：`WebGL`
- 几何内核：`occt-draw-core` Wasm
- 草图、约束、参数、命令系统：TypeScript
- 重计算和内核调用：Web Worker
- 云端能力：通过前端 `cloud-client` 包隔离项目、文件、版本和协同接口

## 分包结构

```txt
apps/
  web/                    # React + Vite 三维云端 CAD 工作台

packages/
  shared/                 # 最底层通用类型、常量和工具
  math/                   # 三维数学、几何基础量、容差和坐标计算
  protocol/               # Worker、Wasm、云端任务的消息协议
  core/                   # CAD 文档、命令、选择、撤销重做、装配领域核心
  sketch/                 # 草图平面、草图元素和草图工具流
  constraints/            # 草图约束、尺寸约束和约束管理
  parametrics/            # 参数、表达式、变量表和特征驱动关系
  scene/                  # 三维场景图、对象树、显示状态和选择集
  renderer/               # 渲染引擎抽象接口
  renderer-webgl/         # WebGL 渲染后端
  wasm-bridge/            # occt-draw-core Wasm 的唯一桥接层
  worker-client/          # 主线程侧 Worker 客户端封装
  worker-runtime/         # Worker 侧任务调度和 Wasm 调用编排
  cloud-client/           # 云端项目、文件、版本、权限和协同客户端
  ui/                     # 自有 CAD 工作台组件库
  config/                 # 工程配置沉淀
```

## 分层职责

`shared` 是最底层公共包，只放稳定、无业务方向的通用能力。

`math` 是 CAD 前端数学基础包，后续承载向量、矩阵、四元数、射线、平面、包围盒、容差比较和坐标变换。业务层不应各自散写数学工具。

`core` 负责 CAD 领域核心，不依赖 React、WebGL、Worker、Wasm 细节。它面向文档、命令、选择、历史、装配和特征关系。

`sketch`、`constraints`、`parametrics` 都用 TypeScript 实现。草图和约束是编辑器逻辑，不放进 Wasm 内核里；Wasm 主要负责高精度几何和拓扑计算。

`scene` 是渲染和工作台之间的三维显示模型，管理对象树、可见性、选择集和显示状态。

`renderer` 定义渲染抽象，`renderer-webgl` 提供 WebGL 实现。上层工作台不直接依赖 WebGL 细节。

`wasm-bridge` 是唯一知道 `occt-draw-core` Wasm 加载、初始化、调用和错误收敛细节的包。

`worker-client` 和 `worker-runtime` 隔离 Worker 通信细节。页面层只调用稳定客户端，不直接写 `postMessage` 协议。

`cloud-client` 隔离云端 API，后续承载项目、文件、版本、权限、协同和云任务状态。

## 依赖方向

依赖方向必须保持单向、可解释，并由 `dependency-cruiser` 自动检查。

```txt
shared
math -> shared
protocol -> shared
core -> math, shared
sketch -> core, math, shared
constraints -> sketch, core, math, shared
parametrics -> core, shared
scene -> core, math, shared
renderer -> scene, math, shared
renderer-webgl -> renderer, scene, math, shared
wasm-bridge -> protocol, shared
worker-client -> protocol, shared
worker-runtime -> wasm-bridge, protocol, shared
cloud-client -> protocol, shared
apps/web -> 组合所有面向运行时的包
```

## Wasm 边界

Wasm 是几何内核，不是前端编辑器状态层。

前端 TypeScript 负责：

- 草图工具流
- 草图元素管理
- 约束模型和约束状态
- 参数和表达式
- 命令系统
- 撤销重做
- 选择集
- 场景显示状态

Wasm 负责：

- 高精度几何计算
- 拓扑生成
- 布尔运算
- 特征求值
- 曲线曲面计算
- 模型网格化或几何输出

推荐调用链：

```txt
apps/web
  -> worker-client
    -> worker-runtime
      -> wasm-bridge
        -> occt-draw-core Wasm
```

## 云端 CAD 特有边界

三维云端 CAD 不只是本地三维编辑器，需要提前为这些能力留边界：

- 用户、组织和权限
- 项目和文件空间
- 模型版本
- 增量保存
- 协同编辑
- 云端几何任务
- 模型转换和导入导出任务

这些能力先进入 `cloud-client` 和 `protocol` 的边界，不直接塞进页面组件。

## 当前阶段目标

当前阶段以 [路线图](./路线图.md) 的第 1 阶段为准：CAD 工作台骨架与基础交互。

第 0 阶段的工程底座已经基本完成，当前重点是让下面这些边界稳定下来：

1. `core` 中的文档、PartStudio、对象、选择目标和特征占位。
2. `scene` 中的领域对象到渲染对象映射。
3. `renderer` 和 `renderer-webgl` 中的相机、视图、拾取、线框渲染和高亮输入。
4. `apps/web` 中的工作台布局、命令入口、选择联动、模型树和属性面板。
5. 后续草图系统进入前，需要先补齐命令生命周期、选择边界和撤销重做最小模型。

当前阶段仍不实现真实草图、真实拉伸、Wasm 几何内核和云端保存。
