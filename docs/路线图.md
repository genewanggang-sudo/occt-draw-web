# occt-draw-web 路线图

本文档是 `occt-draw-web` 后续开发的主线约束。新增功能、重构和文档修改都应先确认是否服务于当前阶段；如果任务无法归入当前阶段，需要先更新本路线图或明确推迟。

## 产品方向

`occt-draw-web` 的目标是浏览器中的三维云端 CAD 工作台。

长期能力包括：

- 三维 CAD 工作台：文档、零件工作室、模型树、属性面板、命令系统和视窗交互。
- 三维场景与渲染：WebGL 渲染后端、相机控制、选择高亮、后续实体和网格显示。
- 草图与参数化：草图元素、尺寸、约束、参数和特征历史。
- 几何内核接入：通过 Worker 和 `wasm-bridge` 调用 `occt-draw-core` Wasm。
- 云端能力：项目、文件、版本、权限、协同和云端任务接口。

不做的事情：

- 不在前端 TypeScript 中实现 B-Rep、布尔、倒角、圆角等底层几何内核。
- 不把 Wasm 当作前端编辑器状态层。
- 不回到旧的二维出图编辑器方向。
- 不为了演示效果提前堆叠跨阶段功能。

## 当前阶段

当前处于 **第 1 阶段收口，准备进入第 2 阶段草图系统 v1**。

第 0 阶段“技术底座收口”已经基本完成：monorepo、工具链、包边界、WebGL 最小渲染、三维视图导航、默认 CAD 文档、场景映射、基础命令和选择链路已经存在。

当前阶段的目标不是正式建模，而是把产品从“能显示三维视窗”推进到“具备 CAD 工作台的稳定骨架”。当前命令、选择、视图、模型树、属性面板和主要包边界已经基本收口，下一步可以进入草图系统 v1。

## 当前实现快照

截至当前代码，已经实现：

- `apps/web` React + Vite 工作台入口。
- 顶部工作台、命令工具栏、视图工具栏、左侧模型树、右侧属性面板和中央视窗。
- 默认 `CadDocument`、`PartStudio`、基准网格、坐标轴和调试立方体。
- `core` 中的文档、对象、选择目标和 PartStudio 基础领域模型。
- `math` 中的三维向量和基础数学函数。
- `scene` 中的 CAD 对象到渲染场景对象映射。
- `renderer` 中的相机、包围盒、包围球、渲染输入和拾取抽象。
- `renderer-webgl` 中的 WebGL 线框渲染、基础 shader、视图投影矩阵和高亮状态。
- 视图导航：旋转、平移、缩放、适配视图、标准视图。
- 基础选择：线框立方体边拾取、对象选择、预选、高亮、空白清空选择、模型树联动和属性面板展示。
- 命令生命周期 v1：`select`、`sketch`、`extrude` 入口、命令状态、禁用原因和属性面板反馈，其中 `extrude` 当前禁用。
- `core`、`scene`、`renderer-webgl` 已完成包内职责拆分，`src/index.ts` 只作为公共 API 门面。
- 工程护栏：TypeScript strict、ESLint、Prettier、type-coverage、dependency-cruiser、husky、commitlint、CI。

尚未实现：

- 真实草图平面和草图绘制。
- 草图对象数据模型。
- 尺寸和几何约束。
- 参数表达式和特征驱动关系。
- 针对真实文档编辑的撤销重做历史。
- Wasm 几何内核真实加载和 Worker 调度。
- 实体建模、拉伸和特征重算。
- 文档保存、打开、云端同步和权限协作。

## 阶段路线

### 第 0 阶段：技术底座收口

状态：基本完成。

目标：

- 建立 monorepo、工具链、包边界和 CI 护栏。
- 建立三维 CAD 的包结构和依赖方向。
- 建立 WebGL 最小三维视窗。
- 建立默认文档、场景映射和最小工作台。

已完成：

- `pnpm workspace`、TypeScript strict、ESLint、Prettier、type-coverage、dependency-cruiser。
- `apps/web` 和 `packages/*` 基础结构。
- WebGL 线框渲染后端。
- 正交相机、标准视图、旋转、平移、缩放和适配视图。
- 默认文档、模型树、属性面板和基础命令入口。

收尾项：

- 文档与当前代码状态对齐。
- 后续所有开发以本文档为阶段依据。

### 第 1 阶段：CAD 工作台骨架与基础交互

状态：基本完成。

目标：

- 让页面具备稳定 CAD 工作台结构。
- 让文档、视窗、命令、选择、模型树和属性面板形成闭环。
- 为第 2 阶段草图系统提供明确入口。

已完成：

- 命令生命周期 v1：进入命令、取消命令、命令状态、禁用原因和命令面板反馈。
- 选择系统边界 v1：对象选择、边选择、预选、高亮、空白清空选择、模型树联动和属性展示。
- `core` 文档模型初步固化：`CadDocument`、`PartStudio`、`Feature`、`CadObject` 和选择目标已有清晰包内模块。
- `scene` 映射边界初步固化：只把领域对象转换为渲染场景对象，不承载业务状态。
- `renderer-webgl` 包内职责初步固化：生命周期、渲染管线、几何生成、矩阵和 shader 分离。
- 工作台 UI 状态已能展示当前命令、禁用原因、选中对象、当前文档和当前 PartStudio。

剩余收口项：

1. 保持文档与当前代码状态同步。
2. 进入草图系统前，只允许做必要的小型架构收口，不继续扩展工作台壳功能。
3. 撤销重做不在本阶段实现；选择、相机、命令高亮属于临时 UI 状态，不进入建模历史。撤销重做等草图或特征产生真实文档编辑后再做。

完成标准：

- 页面打开后能稳定展示默认 CAD 文档。
- 用户能切换基础命令，并清楚看到当前命令状态。
- 用户能选择三维对象或子元素，并在模型树和属性面板看到一致结果。
- 选择和命令状态经过统一控制器流转，不散落在 React 组件里。
- 工作台骨架允许后续接入草图命令，而不需要重写应用状态结构。
- 包内实现不继续堆叠到 `src/index.ts`，公共 API 通过门面显式导出。

本阶段不做：

- 不实现真实草图绘制。
- 不实现真实拉伸建模。
- 不接入真实 Wasm 几何内核。
- 不做云端保存和协同。
- 不为选择、相机或命令高亮单独实现撤销重做。

### 第 2 阶段：草图系统 v1

状态：下一步开始。

目标：

- 从工作台进入草图编辑状态。
- 允许在指定平面上创建最小草图几何。
- 草图数据进入 CAD 文档模型，并能通过场景层显示。

开发顺序：

1. 定义默认草图平面和草图坐标系，第一版先使用默认 XY 平面。
2. 定义最小草图对象模型：草图、点、线段。
3. 实现进入和退出草图命令。
4. 实现线段绘制最小闭环：点击起点，再点击终点。
5. 将草图线段投影到 `scene`，并由 WebGL 显示。
6. 在属性面板显示草图状态和基础对象信息。

完成标准：

- 用户能进入一个默认草图平面。
- 用户能绘制至少一条线段，并保留线段端点数据。
- 草图数据保存在 `CadDocument` 中。
- 离开草图后仍能看到草图几何。
- 草图命令和选择系统可以共同工作。

本阶段不做：

- 不做复杂约束求解。
- 不做圆、圆弧和复杂草图工具。
- 不做拉伸生成实体。
- 不做草图文件保存。

### 第 3 阶段：约束与尺寸 v1

状态：未开始。

目标：

- 让草图从自由绘制进入基础 CAD 草图状态。

开发顺序：

1. 定义约束和尺寸数据模型。
2. 实现重合、水平、垂直、平行、垂直约束。
3. 实现距离、半径、角度尺寸占位。
4. 实现轻量约束求解器雏形。
5. 支持拖动草图点并保持基础约束。
6. 在 UI 中显示约束和尺寸状态。

完成标准：

- 草图元素可以被基础约束驱动。
- 尺寸和约束作为文档数据保存。
- 约束失败或过约束状态可以被表达。

本阶段不做：

- 不追求工业级完整求解器。
- 不引入 Wasm 承担草图约束求解。

### 第 4 阶段：参数与特征前置模型

状态：未开始。

目标：

- 建立参数、表达式、特征树和重算队列的 TypeScript 侧模型。
- 为 Wasm 拉伸和后续特征建模准备输入输出边界。

开发顺序：

1. 定义参数表和表达式求值边界。
2. 定义 Feature timeline。
3. 定义 Sketch feature。
4. 定义 Extrude feature 的 TypeScript 输入模型。
5. 建立特征重算状态：ready、dirty、failed、suppressed。
6. 建立错误展示和失败恢复路径。

完成标准：

- 模型树能显示特征历史。
- 参数变化可以标记相关特征需要重算。
- Extrude feature 已有完整前端数据结构，但不一定调用真实内核。

### 第 5 阶段：Wasm 几何内核接入

状态：未开始。

目标：

- 通过 Worker 调用 `occt-draw-core` Wasm，让草图生成三维几何。

开发顺序：

1. 明确 `wasm-bridge` 加载、初始化、错误收敛接口。
2. 明确 `protocol` 中的 Worker 消息。
3. 实现 `worker-client` 主线程调用封装。
4. 实现 `worker-runtime` 任务调度。
5. 实现 profile 识别和拉伸任务。
6. 将内核返回的 mesh 或边线结果转换为 `scene` 可渲染对象。

完成标准：

- 用户能绘制闭合草图并执行拉伸。
- Worker 调用失败可以反馈到特征状态。
- WebGL 能显示拉伸结果的边线或面片。

本阶段不做：

- 不把应用状态放进 Worker。
- 不让页面组件直接调用 Wasm。

### 第 6 阶段：特征建模 v1

状态：未开始。

目标：

- 形成最小可用的参数化建模闭环。

开发顺序：

1. 完善 Sketch feature 和 Extrude feature。
2. 支持 Boolean add、cut、new body 的最小语义。
3. 支持编辑特征参数并触发重算。
4. 支持重算失败、依赖断裂和恢复展示。
5. 支持模型树中特征选择和基础编辑。

完成标准：

- 用户能创建草图、拉伸、修改参数并重新计算。
- 模型树能反映建模历史。
- 特征失败不会破坏整个文档状态。

## 开发节奏规则

- 每次开发前先确认任务属于哪个阶段。
- 当前阶段未完成前，不提前实现后续阶段的复杂功能。
- 可以预留接口，但预留必须服务于当前阶段的完成标准。
- UI 优先服务核心 CAD 流程，不做与当前阶段无关的产品包装。
- Wasm 只负责几何内核和高精度计算，不承载命令、选择、草图交互和应用状态。
- 云端能力先进入 `cloud-client`、`protocol` 和清晰边界，不直接散落在 React 组件中。
- 如果实现发现路线图不合理，先修改本文档，再继续实现。

## 文档维护规则

- 当前阶段变化时，必须更新“当前阶段”和“当前实现快照”。
- 完成阶段验收项时，必须把状态从“进行中”改为“基本完成”或“完成”。
- 新增包或改变依赖方向时，必须同步更新 [架构设计](./架构设计.md) 和 [工程规范](./工程规范.md)。
- 新增运行时资源、环境变量、Wasm 或云端约定时，必须同步更新 [运行时约定](./运行时约定.md)。
