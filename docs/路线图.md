# occt-draw-web 路线图

## 产品方向

`occt-draw-web` 的目标是浏览器中的三维云端 CAD 工作台。

长期能力包括：

- 三维 CAD 工作台：文档、零件工作室、模型树、属性面板、命令系统和视窗交互。
- 草图与参数化：草图元素、尺寸、约束、参数和特征历史。
- 几何内核接入：通过 Worker 和 `wasm-bridge` 调用 `occt-draw-core` Wasm。
- 云端能力：项目、文件、版本、权限、协同和云端任务接口。

不做的事情：

- 不在前端 TypeScript 中实现 B-Rep、布尔、倒角、圆角等底层几何内核。
- 不把 Wasm 当作前端编辑器状态层。
- 不回到二维出图编辑器方向。

## 当前阶段

当前处于 **第 1 阶段收口，准备进入第 2 阶段草图系统 v1**。

第 1 阶段目标是让工作台具备稳定 CAD 骨架：文档、视窗、命令、选择、模型树、属性面板、显示模型投影和 WebGL 渲染已经形成闭环。

本轮基建整改已经把关键边界调整为：

- `math`：class-based CAD 几何对象。
- `core`：`EditDraft / DocumentOperation / DocumentTransaction / TransactionGroup`。
- `display`：`CadDocument + EditDraft -> DisplayModel`。
- `renderer`：渲染、拾取和相机协议。
- `renderer-webgl`：WebGL 绘制实现。
- `apps/web`：`CadCommand class` 命令入口。

## 当前实现快照

- React + Vite 工作台入口。
- 顶部工作台、命令工具栏、视图工具栏、模型树、属性面板和中央视窗。
- 默认 `CadDocument`、`PartStudio`、基准网格、坐标轴和调试立方体。
- WebGL 线框渲染、基础 shader、视图投影矩阵和高亮状态。
- 视图导航：旋转、平移、缩放、适配视图、标准视图。
- 基础选择：立方体边拾取、对象选择、预选、高亮、空白清空选择、模型树联动。
- 命令生命周期：`select`、`sketch`、`extrude` 入口，其中 `extrude` 当前禁用。

尚未实现：

- 真实草图平面和草图绘制。
- 草图对象数据模型。
- 尺寸和几何约束。
- 参数表达式和特征驱动关系。
- 针对真实文档编辑的 undo/redo UI。
- Wasm 几何内核真实加载和 Worker 调度。
- 实体建模、拉伸和特征重算。
- 文档保存、打开、云端同步和权限协作。

## 阶段路线

### 第 0 阶段：技术底座收口

状态：基本完成。

目标：

- 建立 monorepo、工具链、包边界和 CI 护栏。
- 建立 WebGL 最小三维视窗。
- 建立默认文档和最小工作台。

### 第 1 阶段：CAD 工作台骨架与基础交互

状态：基本完成，正在收口。

目标：

- 页面具备稳定 CAD 工作台结构。
- 文档、视窗、命令、选择、模型树和属性面板形成闭环。
- 为第 2 阶段草图系统提供明确入口。

完成标准：

- 页面能稳定显示默认 CAD 文档。
- 用户能切换基础命令并看到当前命令状态。
- 用户能选择三维对象或子元素，并在模型树和属性面板看到一致结果。
- 选择和命令状态经过统一控制器流转，不散落在 React 组件里。
- 包内实现不继续堆到 `src/index.ts`。

### 第 2 阶段：草图系统 v1

状态：下一步开始。

目标：

- 从工作台进入草图编辑状态。
- 在默认平面上创建最小草图几何。
- 草图数据进入 CAD 文档模型，并通过 `display` 显示。

开发顺序：

1. 定义默认草图平面和草图坐标系，第一版使用默认 XY 平面。
2. 定义最小草图对象模型：草图、点、线段。
3. 实现进入和退出草图命令。
4. 实现线段绘制最小闭环：点击起点，再点击终点。
5. 线段绘制过程使用 `EditDraft` 显示临时线段。
6. 确认后生成 `DocumentOperation`，正式写入 `CadDocument`。
7. `display` 把正式草图线段投影为 `DisplayModel`，由 WebGL 显示。

本阶段不做：

- 不做复杂约束求解。
- 不做圆、圆弧和复杂草图工具。
- 不做拉伸生成实体。
- 不做云端保存。
